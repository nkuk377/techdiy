Firefox Browser based on Alpine Linux - Docker image

This Docker image provides a lightweight and secure environment to run the "Firefox Browser" Based on "Alpine Linux", it is designed for users who prioritize speed and efficiency while maintaining privacy and security while browsing the web.

Requirements:

1. Docker: Ensure Docker is installed on your system.

2. X11 Server: You need to have an X11 server running for the graphical interface.

Usage:

👣 Create a folder firefox-alpine

👣 Create two files in that folder: dockerfile & run-firefox.sh with the code below...

dockerfile: 
# Use the Alpine base image
FROM alpine:latest

# Install Firefox, X11, and other necessary dependencies
RUN apk update && apk add --no-cache \
    firefox \
    dbus \
    ttf-dejavu \
    font-noto \
    xvfb \
    xauth \
    bash \
    ffmpeg \
    pulseaudio \
    mesa-gl \
    dbus-glib

# Install glibc from sgerrand, ensuring there is no conflict with nsswitch.conf
RUN apk --no-cache add --virtual .build-deps curl ca-certificates && \
    curl -Lo /etc/apk/keys/sgerrand.rsa.pub https://alpine-pkgs.sgerrand.com/sgerrand.rsa.pub && \
    curl -Lo glibc-2.35-r0.apk https://github.com/sgerrand/alpine-pkg-glibc/releases/download/2.35-r0/glibc-2.35-r0.apk && \
    # Backup existing nsswitch.conf to avoid conflict
    cp /etc/nsswitch.conf /etc/nsswitch.conf.bak || true && \
    apk add --force-overwrite glibc-2.35-r0.apk && \
    # Restore the original nsswitch.conf if needed
    cp /etc/nsswitch.conf.bak /etc/nsswitch.conf || true && \
    rm -f glibc-2.35-r0.apk && \
    apk del .build-deps

# Add a non-root user for running Firefox
RUN adduser -D nonrootuser

# Switch to the non-root user
USER nonrootuser

# Set the environment variable to forward the X display
ENV DISPLAY=:99

# Create a startup script to run Firefox with Xvfb
RUN echo -e '#!/bin/bash\n\
Xvfb :99 -screen 0 1280x1024x16 &\n\
firefox' > /home/nonrootuser/start-firefox.sh

# Make the script executable
RUN chmod +x /home/nonrootuser/start-firefox.sh

# Set the entrypoint to start the script
ENTRYPOINT ["/bin/bash", "/home/nonrootuser/start-firefox.sh"]
run-firefox.sh: 
#!/bin/bash

# Build the Docker image for Firefox Browser if it doesn't already exist
docker build -t dockerfos/firefox-alpine .

# Ensure the X11 socket is shared with the container
# Allow local docker to access the X server
xhost +local:docker

# Set up PulseAudio server and socket paths for the sound!
PULSE_SERVER="unix:/run/user/$(id -u)/pulse/native"
PULSE_SOCKET="/run/user/$(id -u)/pulse/native"
PULSE_COOKIE="$HOME/.config/pulse/cookie"

# Run the Firefox container with X11 forwarding
docker run -it \
  -e DISPLAY=$DISPLAY \
  -e PULSE_SERVER="$PULSE_SERVER" \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  -v "$PULSE_SOCKET:$PULSE_SOCKET" \
  -v "$PULSE_COOKIE:/root/.config/pulse/cookie:ro" \
  dockerfos/firefox-alpine

# Optionally revoke access after the container stops (for security reasons)
echo "Revoking Docker access to X11 server..."
xhost -local:docker

👣 Make run-firefox.sh executable:

chmod +x run-firefox.sh

👣 No need to build the image - just run the script:

./run-firefox.sh

💡 Notes:

• Not for Root Use: This image is not intended to be run as the root user for security reasons!
