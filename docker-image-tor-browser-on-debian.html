Tor Browser based on Debian:bookworm-slim - Docker image

This Docker image provides a secure, isolated environment to run the Tor Browser↗️⁠. Tor Browser is designed to help protect your privacy online by anonymizing your web traffic and shielding your identity from network surveillance and traffic analysis. This image is built on top of "Debian:bookworm-slim" base image, ensuring a stable and secure environment.

Key Features:

• Tor Browser: Pre-installed and configured to run over the Tor network for anonymous browsing.

• Debian Base: Built on top of Debian bookworm-slim image.

• Isolated Environment: Ensures that the Tor Browser runs in a sandboxed environment.

Requirements:

1. Tor browser needs to be downloaded manually, as for example tor-project-linux-x86_64.5.7.tar.xz

No need to install the tor browser via package manager !

2. xhost (Utility to control X server access) needs to be installed if you'll have error about it while running the script.

(See the .sh script, line: `xhost +local:docker`)

Usage:

👣 Create a folder tor-browser-debian

👣 Place downloaded tor browser archive - tor-project-linux-x86_64.5.7.tar.xz into that folder

No need to extract it !

👣 Rename the archive into tor-browser.tar.xz - IMPORTANT!

• Make sure the folder name inside the archive is tor-browser and not for example tor-browser_en_US or so. - IMPORTANT!

if so - you need to rename or extract, rename the folder as tor-browser and re-archive it as tor-browser.tar.xz. - IMPORTANT!

👣 Create two files in the same folder: dockerfile & run-tor-browser.sh with the code below...

dockerfile: 
FROM debian:bookworm-slim

# Install necessary dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    ca-certificates \
    libasound2 \
    libdbus-glib-1-2 \
    libgtk-3-0 \
    libx11-xcb1 \
    libxt6 \
    libpci3 \
    xz-utils \
    ffmpeg \
    pulseaudio \
    apt-utils \
    curl \
    libgl1-mesa-glx \
    libegl1-mesa \
    && rm -rf /var/lib/apt/lists/*

# Set the working directory
WORKDIR /home/torbrowser

# Copy the pre-downloaded tor-browser.tar.xz into the Docker image
COPY tor-browser.tar.xz /home/torbrowser/

# Extract the Tor Browser tarball
RUN tar -xf tor-browser.tar.xz && \
    rm tor-browser.tar.xz

# Set the correct permissions for the torbrowser user
RUN useradd -ms /bin/bash torbrowser
RUN chown -R torbrowser:torbrowser /home/torbrowser

# Switch to the torbrowser user
USER torbrowser
WORKDIR /home/torbrowser/tor-browser

# Run the Tor Browser (without starting a new X server inside the container)
ENTRYPOINT ["./start-tor-browser.desktop", "--verbose"]
run-tor-browser.sh: 
#!/bin/bash

# Build the Docker image for Tor Browser if it doesn't already exist
docker build -t dockerfos/tor-browser-debian .

# Set the absolute path to your Tor Browser folder on the host
TOR_BROWSER_DIR="$HOME/'dockerfos/tor-browser-debian'"

# Allow Docker to access the X server for graphical applications
echo "Granting local Docker access to X11 server..."
xhost +local:docker

# Check if DISPLAY variable is set (for X11 forwarding)
if [ -z "$DISPLAY" ]; then
    echo "DISPLAY variable is not set. Please ensure X11 is running."
    exit 1
fi

# Check if the Docker image 'dockerfos/tor-browser-debian' exists
if [[ "$(docker images -q dockerfos/tor-browser-debian 2> /dev/null)" == "" ]]; then
    echo "Docker image 'dockerfos/tor-browser-debian' not found. Building the image..."
    
    # Build the Docker image using the pre-downloaded Tor Browser tarball
    docker build -t dockerfos/tor-browser-debian .
    
    if [ $? -ne 0 ]; then
        echo "Error occurred while building the Docker image."
        exit 1
    fi
else
    echo "Docker image 'dockerfos/tor-browser-debian' found."
fi

# Set up PulseAudio server and socket paths for the sound!
PULSE_SERVER="unix:/run/user/$(id -u)/pulse/native"
PULSE_SOCKET="/run/user/$(id -u)/pulse/native"
PULSE_COOKIE="$HOME/.config/pulse/cookie"

# Run the Docker container for Tor Browser
echo "Running Tor Browser inside Docker..."
docker run -e DISPLAY=$DISPLAY \
    -e PULSE_SERVER="$PULSE_SERVER" \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -v "$PULSE_SOCKET:$PULSE_SOCKET" \
    -v "$PULSE_COOKIE:/root/.config/pulse/cookie:ro" \
    dockerfos/tor-browser-debian

# Optionally revoke access after the container stops (for security reasons)
echo "Revoking Docker access to X11 server..."
xhost -local:docker

👣 Make run-tor-browser.sh executable:

chmod +x run-tor-browser.sh

👣 No need to build the image - just run the script:

./run-tor-browser.sh

💡 Security Considerations:

• This container is designed to run in a sandboxed environment with minimal exposure to the host system.

• It's recommended to run this image using non-privileged user permissions and limit access to sensitive directories.
